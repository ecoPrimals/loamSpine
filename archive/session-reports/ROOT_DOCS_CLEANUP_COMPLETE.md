## Maintenance Guidelines

### When to Add Files to Root

**✅ Add to root** if:
- Essential for daily development
- Required for production deployment
- Needed by external users/contributors
- Part of current release documentation

**❌ Don't add to root** if:
- Historical/completed session reports
- Internal planning documents
- Temporary status updates
- Superseded by newer versions

### When to Archive

**Archive immediately** when:
- Document becomes historical
- Replaced by newer version
- Session/audit complete
- No longer actively referenced

**Archive to**:
- `archive/audit-reports/` → Comprehensive audits
- `archive/session-reports/` → Development sessions, planning, status
- `archive/showcase-reports/` → Showcase-specific reports (if any)

### Naming Convention

**Active documents**:
- `README.md` (permanent)
- `STATUS.md` (permanent, continuously updated)
- `RELEASE_NOTES_v0.7.0.md` (version-specific)
- `DEPLOYMENT_GUIDE_v0.7.0.md` (version-specific)

**Archive documents**:
- `*_DEC_28_2025.md` (date-stamped session reports)
- `*_COMPLETE.md` (completed tasks)
- `*_PLAN.md` (planning docs after execution)

## 🎓 Maintenance Guidelines

### When to Add to Root

**✅ Yes** if:
- Essential for daily development
- Required for deployment
- Needed by external users
- Current release documentation

**❌ No** if:
- Historical/completed
- Internal planning
- Temporary status
- Superseded versions

### When to Archive

**Archive when**:
- Document becomes historical
- Replaced by newer version
- Session/audit complete
- No longer actively referenced

**Archive to**:
- `archive/audit-reports/` → Audits
- `archive/session-reports/` → Sessions, planning
- Keep git history (rename, not delete)
